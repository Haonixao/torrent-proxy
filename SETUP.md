# Инструкция по настройке TorrentProxy

В данном документе приведено детальное описание конфигурационных файлов для клиента и сервера TorrentProxy.

---

## 1. Схема конфигурации Клиента (`client/config.yaml`)

Конфигурация клиента определяет параметры локального SOCKS5-прокси и способ подключения к удаленному серверу.

### Структура параметров:

| Параметр | Тип | Описание | Значение по умолчанию |
| :--- | :--- | :--- | :--- |
| `listen` | string | Адрес и порт, на которых будет запущен локальный SOCKS5 сервер. | `"0.0.0.0:1080"` |
| `server` | string | Адрес удаленного TorrentProxy сервера. Поддерживает одиночные порты и диапазоны (Port Hopping). | **Обязательно** (напр. `"ip:50000-50100"`) |
| `udp` | bool | Включение поддержки UDP Associate в SOCKS5. | `true` |
| `decoy-traffic` | bool | Включение модуля фонового «отвлекающего» трафика к мировым ресурсам. | `false` |
| **`torrent`** | **struct** | **Настройки протокола BitTorrent (L7)** | |
| `  auth-key` | string | Ключ авторизации (64 hex символа / 32 байта). Должен совпадать с серверным. | **Обязательно** |
| `  info-hash` | string | 20-байтный хеш (40 hex символов), имитирующий конкретный торрент. | Генерируется случайно, если пуст |
| `  sessions-num` | int | Количество параллельных uTP-сессий (туннелей) до сервера. | `5` |
| `  connections-time-out` | int | Таймаут простоя соединений в секундах. | `60` |

### Пример `config.yaml` клиента:

```yaml
listen: "127.0.0.1:1080"
server: "your_vps_ip:50000-50100"
udp: true
decoy-traffic: true

torrent:
  auth-key: "0270d4b40594112884add073c8296b39c9e1effbb56d50476bcb7e816c095ec5"
  info-hash: "62a4d9e139f3315f8716bcccca0cc984a9809da1"
  sessions-num: 5
  connections-time-out: 60
```

## 2. Схема конфигурации Сервера (`server/config.yaml`)

Серверная часть отвечает за прием uTP-соединений и активную мимикрию.

### Структура параметров:

| Параметр | Тип | Описание | Значение по умолчанию |
| :--- | :--- | :--- | :--- |
| `port` | string | Порт или диапазон портов (UDP), на которых сервер будет ожидать подключения. | `"50000-50100"` |
| **`torrent`** | **struct** | **Настройки протокола BitTorrent (L7)** | |
| `  auth-key` | string | Ключ авторизации (64 hex символа). Должен совпадать с клиентским. | **Обязательно** |
| `  info-hash` | string | Хеш торрента (40 hex символов). Для максимальной маскировки должен совпадать с клиентским. | `""` |

### Пример `config.yaml` сервера:

```yaml
port: "50000-50100"

torrent:
  auth-key: "0270d4b40594112884add073c8296b39c9e1effbb56d50476bcb7e816c095ec5"
  info-hash: "62a4d9e139f3315f8716bcccca0cc984a9809da1"
```

---

## 3. Особенности настройки

1.  **Port Hopping**: Если в параметре `server` (у клиента) или `port` (у сервера) указан диапазон (напр. `50000-50100`), клиент будет случайно выбирать порт при каждой ротации сессии. Сервер при этом слушает *весь* указанный диапазон одновременно.
2.  **Безопасность**: Никогда не используйте дефолтный `auth-key` в публичных сетях. Генерируйте свой (например, через `openssl rand -hex 32`).
3.  **Info-Hash**: Для лучшей маскировки рекомендуется найти Info-Hash реально существующего популярного торрента (например, дистрибутива Ubuntu) и прописать его в оба конфига.
